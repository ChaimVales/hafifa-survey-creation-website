6הגדרת TypeScript - tsconfig.json

📌 יצרנו קובץ קונפיגורציה של TS עם הגדרות נכונות לשרת Node.

⌨️ פקודה (יוצרת קובץ אוטומטית)

📄 הקובץ הסופי שלנו: tsconfig.json

{
  "compilerOptions": {

🔍 מה כל שדה אומר

שדהמשמעות
target: "ES2022"קמפל לגרסת JavaScript של 2022
module: "commonjs"סגנון require/module.exports - הסטנדרט של Node
rootDir: "./src"הקוד שלי נמצא ב-src/
outDir: "./dist"שים את הקוד המקומפל ב-dist/
esModuleInterop: trueמאפשר import express from 'express'
strict: trueקפדנות בטיפוסים - תופס באגים מוקדם

למה צריך את הקובץ? בלעדיו, TypeScript לא יודע איך לקמפל - לאיזו גרסת JS, מאיפה לקרוא קוד, לאן לשמור. זו "תעודת הזהות" של TS בפרויקט.

קודמה הוא עושה
import express from 'express'מייבא את הספרייה (סגנון מודרני - אפשרי בזכות esModuleInterop)
const app = express()יוצר אפליקציית Express חדשה
app.get('/', ...)מגדיר נקודת קצה: בקשה GET ל-"/" → מחזיר "Hello from Server!"

למה import ולא require? סגנון import הוא הסטנדרט של ES Modules (חדש). require הוא של CommonJS (ישן). ב-TypeScript מועדף import - יותר נקי וקיבל טיפוסים מלאים.

12התקנת mongoose וחיבור ל-MongoDB

📌 התקנו את mongoose, ועדכנו את index.ts לחיבור ל-MongoDB עם משתני סביבה.

⌨️ פקודה

📄 העדכון העיקרי ב-src/index.ts

import mongoose from 'mongoose'

const port = process.env.PORT || 3000
const mongoUri = process.env.MONGO_URI || 'mongodb://localhost:27017/surveys'

async function start() {
    try {
        await mongoose.connect(mongoUri)
        console.log('Connected to MongoDB')
        app.listen(port, () => console.log(`Listening on ${port}`))
    } catch (err) {
        console.error('MongoDB connection failed:', err)
        process.exit(1)
    }
}
start()

🔍 הסבר השינויים העיקריים

שורהמשמעות

|| 'mongodb://localhost...'ערך ברירת מחדל אם המשתנה לא מוגדר

14User Model + Endpoints ראשונים

📌 יצרנו את User Model עם Mongoose + 2 endpoints (POST, GET) שעובדים מול MongoDB.

📄 קובץ: server/src/models/User.ts

import mongoose, { Schema } from 'mongoose'

const userSchema = new Schema({
    email: { type: String, required: true, unique: true, lowercase: true },
    name: { type: String, required: true },
    passwordHash: { type: String, required: true, select: false }
}, {
    timestamps: true
})

export const User = mongoose.model('User', userSchema)

🔍 הסבר על Schema

שדהתפקיד
required: trueחובה - אחרת Mongoose דוחה
unique: trueייחודי - אסור 2 משתמשים עם אותו email
lowercase: trueהמרה אוטומטית לאותיות קטנות
select: falseלא חוזר ב-find רגיל - מסיבות אבטחה
timestamps: trueMongoose מוסיף createdAt ו-updatedAt אוטומטית

📚 מושגים: Schema vs Model

  • Schema = "התבנית" של דוקומנט (איזה שדות יש, מה הטיפוס)
  • Model = ה"כלי" שיודע לעבוד מול MongoDB - יוצר, שולף, מעדכן

15Authentication מלא: bcrypt + JWT + Middleware

📌 הוספנו register, login, requireAuth middleware ו-/me - מערכת אימות מקצועית.

⌨️ התקנות

📚 הספריות

ספרייהתפקיד
bcryptהופך סיסמה ל-hash מוצפן (אי-הפיך). משווה סיסמאות.
jsonwebtokenיוצר tokens חתומים שאי אפשר לזייף. תעודות זהות.
@types/...טיפוסי TypeScript - dev only

📄 הזרימה של Register

app.post('/api/auth/register', async (req, res) => {
    const { email, name, password } = req.body

    // 1. ולידציה
    if (!email || !name || !password) return res.status(400)...

    // 2. בדיקה שאימייל לא קיים
    const existing = await User.findOne({ email })
    if (existing) return res.status(409).json({ error: 'Email already in use' })

    // 3. הצפנת הסיסמה
    const passwordHash = await bcrypt.hash(password, 10)

    // 4. יצירת המשתמש
    const user = await User.create({ email, name, passwordHash })

    // 5. יצירת token
    const token = jwt.sign({ userId: user._id }, jwtSecret, { expiresIn: '7d' })

    // 6. החזרה (ללא passwordHash!)
    res.status(201).json({ token, user: { id: user._id, email, name } })
})

📄 הזרימה של Login

app.post('/api/auth/login', async (req, res) => {
    const { email, password } = req.body

    // 1. שליפת user עם +passwordHash
    const user = await User.findOne({ email }).select('+passwordHash')
    if (!user) return res.status(401).json({ error: 'Invalid credentials' })

    // 2. השוואה
    const isValid = await bcrypt.compare(password, user.passwordHash)
    if (!isValid) return res.status(401).json({ error: 'Invalid credentials' })

    // 3. token חדש
    const token = jwt.sign({ userId: user._id }, jwtSecret, { expiresIn: '7d' })

    res.json({ token, user: { id: user._id, email, name: user.name } })
})

📄 ה-Middleware - requireAuth

function requireAuth(req, res, next) {
    const header = req.headers.authorization
    if (!header || !header.startsWith('Bearer ')) {
        return res.status(401).json({ error: 'No token provided' })
    }

    const token = header.replace('Bearer ', '')
    try {
        const decoded = jwt.verify(token, jwtSecret) as { userId: string }
        req.user = { id: decoded.userId }
        next()
    } catch {
        return res.status(401).json({ error: 'Invalid token' })
    }
}

// שימוש:
app.get('/api/auth/me', requireAuth, async (req, res) => {
    const user = await User.findById(req.user.id)
    res.json(user)
})

🔍 הקסם של JWT

ה-token מורכב מ-3 חלקים: Header.Payload.Signature

  • Header: איזה אלגוריתם משמש לחתימה
  • Payload: מידע על המשתמש (userId, expiresAt) - לא מוצפן! רק חתום.
  • Signature: חתימה של השרת - מי שינסה לזייף ייחשף

אפשר לראות את התוכן ב-jwt.io - הדבק את ה-token ותראה.

🧪 בדיקות ב-REST Client

אבטחה!

📄 דוגמה: services/auth.service.ts (לוגיקה)

export async function register(input) {
    const existing = await User.findOne({ email: input.email })
    if (existing) throw new EmailExistsError()

    const passwordHash = await bcrypt.hash(input.password, 10)
    const user = await User.create({ ...input, passwordHash })
    const token = jwt.sign({ userId: user._id }, env.jwtSecret, { expiresIn: '7d' })

    return { token, user: { id: user._id, email: user.email, name: user.name } }
}

export async function register(req, res) {
    try {
        const result = await authService.register(req.body)
        res.status(201).json(result)
    } catch (err) {
        if (err instanceof EmailExistsError) {
            return res.status(409).json({ error: err.message })
        }
        res.status(400).json({ error: err.message })
    }
}

// ProtectedRoute - עוטף דפים מוגנים
export default function ProtectedRoute({ children }) {
    const { user, loading } = useAuth()
    if (loading) return <CircularProgress />
    if (!user) return <Navigate to="/login" />
    return <>{children}</>
}

// App.tsx - שימוש

    <ProtectedRoute><Dashboard /></ProtectedRoute>

// services/surveys.service.ts
export async function browseSurveys(filters: BrowseFilters = {}) {
    const query: any = { isActive: true }

    if (filters.search) {
        query.$or = [
            { title: { $regex: filters.search, $options: 'i' } },
            { description: { $regex: filters.search, $options: 'i' } }
        ]
    }

    let sortQuery: any = { createdAt: -1 }
    if (filters.sortBy === 'oldest') sortQuery = { createdAt: 1 }
    if (filters.sortBy === 'title') sortQuery = { title: 1 }

    return Survey.find(query)
        .select('slug title description questions createdAt')
        .sort(sortQuery)
        .limit(100)
}

🔍 ה-route ב-router.get('/browse', ...) חייב להיות לפני /:slug כי express מסדר routes לפי הסדר!

🐳 Multi-stage Dockerfile של ה-Client

# Stage 1: Build React
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# Stage 2: nginx serves the build
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

🔍 הקסם: Multi-stage

  • שלב 1 (builder): Node + npm + tsc + vite. בונה את ה-React build (תיקיית dist)
  • שלב 2 (nginx): רק nginx + הקבצים הסטטיים + config
  • גודל סופי: ~25MB (במקום 300MB+ אם היינו משאירים node)

📄 client/nginx.conf - מאחד הכל

server {
    listen 80;
    server_name localhost;

    # /api/* → server (Express)
    location /api/ {
        proxy_pass http://server:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # Everything else → React SPA
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
